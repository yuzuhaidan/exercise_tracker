# Exercises Tracker

This is a full stack MERN app that allows a user to record, update, and delete exercises.

I learned the skills needed to write a Single Page Application where the frontend used React while the backend used Node and Express. Together with MongoDB, I was able to see my CRUD operations working in real time in an HTML table.

My key takeaways include a long list of experiences that I had while completing the app's functionality:

* understanding what a port is for the React app and why it's important to be on a specific port
* how useful running an app in a development build is (utilizing npm create vite and using npm run dev)
* how creating diagrams like a component tree can keep component logic clear during development
* how the historical XMLHttpRequest has been replaced with the Javascript fetch() API so that web browsers can make HTTP requests to web servers
* ensuring parent to child component relationships use top-down data flow when appropriate, and why this means that a "state" can be shared
* knowing when to lift a state
* debugging args when incorrectly passed to certain functions when constructing certain paths
* applying appropriate separation to code between Model, View, and Controller layers

My future plans for this project are to deploy this MERN app on a cloud service. This will require a switch of the backend's use of MongoDB.
